# DeskTodo – 产品文档（PRD）v2.0

**目标系统**：Windows 11  
**产品定位**：Win11 风格、极低占用、桌面融合的轻量待办 + 便签工具。悬挂在桌面，最小化到托盘，支持鼠标穿透，操作图标仅在悬停时显示。

---

## 一、产品概览

| 项目 | 说明 |
|------|------|
| 产品名称 | DeskTodo（待办 + 便签） |
| 版本 | 2.0 |
| 技术栈 | Tauri 2.x (Rust) + React 18 + TypeScript + Tailwind CSS + Zustand + @tanstack/react-virtual |
| 安装形态 | 安装包 ≤ 5MB，同时提供便携版 zip |
| 运行环境 | Windows 11，多显示器，高 DPI |
| 核心界面 | 亚克力模糊背景、8px 圆角、无边框、自绘标题栏、支持暗黑模式 |
| 资源目标 | 主窗口内存 30–80MB，托盘态 ≤ 10MB；每个额外便签窗口内存 ≤ 20MB；CPU 接近 0% |

---

## 二、全局交互模型

DeskTodo 具有两个核心视图模式：
- **待办主窗口**：结构化任务列表，支持子任务、颜色标签、拖拽排序、历史记录等。
- **便签窗口**：自由浮动的独立笔记卡片，模仿 Win7 便签体验，可多开。

两者共用系统托盘、数据备份引擎、亚克力视觉语言，并统一受桌面集成特性管控。

## 三、待办主窗口

### 3.1 标题栏与固定工具栏
- 窗口顶部存在自定义标题栏，高度 36px，从左至右包含：
  - 窗口标题（可自定义，默认“待办”）
  - **图钉按钮（固定开关）**：切换窗口是否“钉在桌面”。钉住后自动开启全窗口鼠标穿透（除工具栏外），窗口不随 Win+D 隐藏。图标为图钉样式，钉住时显示“钉入”状态。
  - **置顶按钮**：独立控制窗口置顶/取消置顶，与固定状态无关。置顶时图标高亮。
  - 关闭按钮（最小化至托盘）。
- 主内容区域在非固定状态下遵循原有悬停交互；固定状态下仅有工具栏可交互，内容不可操作且鼠标透过。

### 3.2 待办核心功能

#### F1 新建待办
- `⊕` 加号按钮位于标题栏下方或列表顶部，点击后在列表顶部插入空白行并自动聚焦。
- `Enter` 确认；`Esc` 取消并移除空白行。
- 父任务展开后，子任务区域同样提供 `⊕` 新建子任务。

#### F2 完成待办
- 右侧 `✓` 完成图标（悬停行时显示）。点击后：
  - 文本添加删除线，降低透明度，200ms 后移入历史。
  - 底部 toast 提示“已移至历史 [撤销]”，持续 3 秒。支持队列，最多同时显示 2 条。
  - 支持 `Ctrl+Z` 撤销完成。

#### F3 编辑待办
- 双击待办文本原位变为输入框，`Enter` 或失焦保存，`Esc` 取消。
- 单击判定需延迟 300ms 以区分双击和展开/收起。

#### F4 子任务与层级
- 父任务前有展开/收起指示图标，单击文字或图标在下展开子任务列表（再次单击收起）。
- 子任务仅支持一层，不可嵌套。
- 父任务行未完成子任务计数显示为 `[N]`（N 为未完成数），子任务全部完成并移入历史后计数消失。
- 设置中提供开关：“子任务全部完成后自动完成父任务”，默认关闭。

#### F5 拖拽排序
- 左侧 `⋮⋮` 手柄（悬停行时显示），按住拖动排序。
- 子任务只能在其父任务内部排序，不能拖出作用域。
- 拖拽时虚拟滚动锁定位置。

#### F6 颜色标签
- 6 种预设颜色：红 #EF4444、橙 #F97316、黄 #EAB308、绿 #22C55E、蓝 #3B82F6、紫 #A855F7，以及“无颜色”默认。
- 行首 4px 宽彩色竖条始终可见，点击弹出颜色选择器。
- 顶部工具栏提供颜色筛选器（后续迭代启用）。

#### F7 批量操作（多选）
- `Ctrl+点击` 多选，`Shift+点击` 范围选择。
- 选中后出现悬浮工具栏：批量完成、批量改色、批量删除（需确认）。

#### F8 任务备注
- 通过行尾 `…` 图标（悬停显示）或右键任务文字呼出备注编辑区。
- 备注内容以小号文字显示在任务下方，可折叠。数据存放在 JSON 备注字段。

#### F9 快捷键
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+T` | 全局唤起/隐藏主窗口 |
| `Esc` | 关闭主窗口（最小化至托盘） |
| `Ctrl+Z` | 撤销完成操作 |
| `Ctrl+N` | 聚焦新建输入框 |
| `Ctrl+W` | 关闭备注面板 |

### 3.3 主窗口交互规则（IR 更新）

| 规则 | 描述 |
|------|------|
| IR1 | 默认隐藏：拖拽手柄、完成图标、备注图标可见度为隐藏。 |
| IR2 | 悬停浮现：鼠标悬停行时，左侧手柄、右侧完成及备注图标同时 smooth 浮现。 |
| IR3 | 整行反馈：悬停时行背景出现极淡颜色变化。 |
| IR4 （穿透规则） | 固定（穿透）模式下，工具栏以外区域所有操作图标隐藏，不可交互。但内容仍可见。用户必须解锁固定才能编辑。 |

### 3.4 历史记录
- 独立面板，按完成时间倒序分组（今天、昨天、本周、更早）。
- 每条显示创建/完成时间戳。支持恢复至待办列表末尾（含子任务）。
- 提供“清空全部”和“归档为文件”功能。

## 四、桌面集成特性（D 模块强化）

### D1 窗口置顶
- 通过置顶按钮或托盘菜单切换，置顶时窗口始终在最前。

### D2 固定在桌面（图钉）
- 点击图钉按钮将窗口“钉”在桌面：窗口不随 Win+D 最小化，始终在桌面图标之下、壁纸之上。
- 实现方式：Rust 端调用 `SetParent` 将窗口绑定到桌面 WorkerW，并移除任务栏条目。
- 钉住的同时**自动开启鼠标穿透**（仅保留标题栏/按钮栏不穿透）。此状态下拖拽手柄等图标隐藏，窗口边框出现虚线提示。
- 解锁图钉后，窗口恢复正常交互模式，并自动关闭穿透。

### D3 鼠标穿透（工具栏除外）
- 穿透作用区域：除固定工具栏外的整个内容区。标题栏中图钉、置顶、关闭按钮始终可点。
- 工具栏区域通过像素级穿透掩码或独立 WebView 实现不穿透。
- 解锁固定后，穿透状态由托盘菜单独立控制（作为临时穿透开关），但与固定状态联动：固定开启时强制穿透且禁用托盘穿透开关。

### D4 透明度调节
- 设置面板提供滑块（10%-100%）和三档预设（全透/半透/不透明）。仅影响窗口背景和亚克力，文字与图标保持清晰。

### D5 系统托盘
- 所有窗口关闭后程序运行于托盘。右键菜单：
  - 显示待办主窗口 / 显示所有便签
  - 新建便签
  - 置顶开关
  - 鼠标穿透开关
  - 退出
- 双击托盘图标恢复上次活跃窗口。

### D6 开机自启
- 设置内开关，静默启动至托盘。

## 五、便签模式（新增模块）

便签窗口是独立、无边框的浮动笔记卡片，每个便签为一个独立的 Tauri 窗口，拥有与主窗口相同的亚克力风格和桌面集成按钮。

### 5.1 便签创建与管理
- **新建**：托盘右键菜单“新建便签”、全局快捷键 `Ctrl+Shift+N`，或从待办项右键“发送到便签”。
- **多实例**：允许多个便签同时存在。每个便签拥有唯一 ID，位置、大小、颜色、内容独立存储。
- **关闭**：关闭便签窗口时，可选择“立即删除”或“移入回收区”（保留 24 小时）。关闭行为可在设置中调整。
- **管理**：托盘“便签列表”子菜单显示所有便签标题，可切换或关闭指定便签。

### 5.2 便签编辑器
- 内容区默认即编辑态，点击任意位置开始输入文本。
- 支持纯文本 + 基本 Markdown 渲染（仅加粗、斜体、下划线），存储格式为 Markdown 或纯文本。
- 提供基本格式栏（可通过双击或选择文字后出现浮动工具条）：**B** / *I* / U。
- 字号三档切换（小/中/大），字体默认为 Segoe UI Variable（可额外提供 Segoe Print 选项）。
- 支持自动换行，窗口缩放时内容自适应。

### 5.3 便签外观
- 与主窗口一致：亚克力模糊背景、8px 圆角、极简阴影。
- 顶部小型工具栏（高度 32px）：
  - 图钉按钮（固定/穿透控制）
  - 置顶按钮
  - 颜色选择器（6 色，点击切换便签色调）
  - 删除按钮（关闭便签）
- 固定模式下，内容区域同样穿透，仅工具栏可交互。

### 5.4 便签数据
- 自动保存：文本修改后 500ms 自动保存至独立 JSON 文件（`%APPDATA%/DeskTodo/notes/<note_id>.json`）。
- 数据结构：
  ```json
  {
    "id": "uuid",
    "createdAt": 1234567890,
    "updatedAt": 1234567890,
    "title": "自动截取首行",
    "content": "Markdown 文本",
    "color": "#3B82F6",
    "fontSize": "medium",
    "position": { "x": 200, "y": 150 },
    "size": { "width": 300, "height": 200 },
    "pinned": false,
    "alwaysOnTop": false
  }
  ```
- 备份策略与待办数据一致（定时 + 退出备份，7 天循环）。

### 5.5 便签回收区
- 删除的便签进入回收区，保留 24 小时后自动清除。
- 回收区界面（可从托盘打开）允许查看、恢复或立即永久删除。

### 5.6 便签性能
- 每个便签窗口在非激活状态下可销毁 WebView（仅保留 Rust 侧窗口框架），激活时重新加载内容。确保打开 5–10 个便签时总内存 < 100MB。
- 便签窗口也支持透明度独立调节（与主窗口分离）。

## 六、数据导入/导出（全局）

- **待办数据**：设置中提供“导出所有待办与历史”按钮，生成 JSON 文件；提供“导入”按钮，支持合并或替换。
- **便签数据**：托盘菜单或便签管理器中提供“导出全部便签”和“导入便签包”。单个便签可通过右键菜单导出为 `.txt` 或 `.md`。
- 导入前均校验 JSON 格式。

---

## 七、界面状态持久化

每次关闭窗口或进入托盘前，保存：
- 主窗口位置、大小、展开的父任务 ID 列表、滚动位置。
- 每个便签的位置、大小、展开状态。
恢复时还原这些状态，并适配当前显示器。

## 九、技术实现摘要

- **桌面固定**：Rust 侧 `SetParent` 绑定 WorkerW，移除任务栏图标（`WS_EX_TOOLWINDOW` 等）。
- **鼠标穿透与工具栏不穿透**：尝试 `set_ignore_cursor_events` + `set_cursor_event_ignoring_rects`（Tauri 2 可能支持）或创建透明穿透主窗口 + 非穿透 Toolbar 子窗口的组合。
- **多便签窗口**：每个便签构建为独立 Tauri webview 窗口，由主进程统一管理。
- **数据存储**：所有本地 JSON 文件通过 Rust 原子写入，定时备份。
