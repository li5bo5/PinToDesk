# 桌面待办轻量工具 – 产品需求文档（PRD）

**文档版本**: 1.0  
**目标系统**: Windows 11  
**产品定位**: Win11 风格、极低占用、桌面融合的轻量待办工具。始终挂在桌面，最小化到托盘，支持鼠标穿透，操作图标仅在悬停时显示。

---

## 一、产品概览

| 项目 | 说明 |
|------|------|
| 名称 | DeskTodo（暂定） |
| 技术栈 | Tauri 2.x (Rust) + React 18 + TypeScript + Tailwind CSS + Zustand + @tanstack/react-virtual |
| 安装形态 | 安装包 3-5MB，同时提供解压即用便携版 zip |
| 运行环境 | Windows 11，支持多显示器及高 DPI |
| 核心界面 | 圆角亚克力背景窗口，无边框，自绘标题栏，支持暗黑模式 |
| 资源目标 | 内存 30-80MB（托盘态 ≤10MB），CPU 接近 0% |

## 二、核心功能模块

### F1 新建待办
- 顶部提供 `⊕` 加号按钮，点击后在列表顶部插入空白行，输入框自动聚焦。
- 输入文本后按 `Enter` 确认创建；按 `Esc` 取消并移出空白行。
- 支持为父任务添加子任务：展开后点击子任务区域内的 `⊕` 按钮，在子任务列表顶部新建空白子项。
- 新建项遵循虚拟滚动，自动滚动至可见区域。

### F2 完成待办
- 每条待办右侧有 `✓` 完成图标（默认隐藏，悬停所在行时显示）。
- 点击完成图标后：任务文本添加删除线 + 透明度降低，200ms 后移至历史记录。
- 完成后底部短暂出现 toast 提示：“已移至历史 [撤销]”，持续 3 秒，点击“撤销”或按 `Ctrl+Z` 可恢复。
- 连续完成多个任务时，toast 采用队列管理，最多同时显示 2 条，每条独立计时。

### F3 编辑待办
- **双击**待办文本原地变为输入框，直接编辑。
- 按 `Enter` 或失焦保存修改；按 `Esc` 取消并恢复原文本。
- 与展开子项操作不冲突：单击判定延迟 300ms 以区分双击。

### F4 子任务管理
- 父任务文本左侧存在展开/收起指示图标（▶），**单击**文字或图标在下方展开子任务列表（再次单击收起）。
- 子任务仅支持一级，不可嵌套。
- 子任务同样支持新建、完成、双击编辑、颜色标记。
- **父任务行信息展示**：当存在未完成的子任务时，父任务文本末尾自动显示子任务计数（如 `[3]` 表示有 3 条未完成子任务）。展开后可见具体条目。
- **父任务自动完成**（可选）：设置中提供开关“子任务全部完成后自动完成父任务”。默认关闭，父任务仍须手动完成。
- 子任务全部完成并移入历史后，父任务行的子任务计数消失。

### F5 拖拽排序
- 左侧拖拽手柄（⋮⋮ 图标），按住拖动可调整顺序。
- 子任务手柄仅在同级范围内排序，不可拖出父任务区域。
- 父任务排序与子任务排序独立。
- 拖拽时虚拟滚动锁定，避免意外滚动。

### F6 颜色标签
- 每条待办可标记一个颜色（预设 6 色：红 #EF4444、橙 #F97316、黄 #EAB308、绿 #22C55E、蓝 #3B82F6、紫 #A855F7），支持“无颜色”默认状态。
- 行首显示 4 px 宽的彩色竖条，颜色始终可见。
- 点击色条弹出小型色块选择器，允许修改或清除颜色。
- 提供颜色筛选视图：点击颜色小圆点（顶部工具栏）仅显示该颜色标签的任务（后续迭代）。

### F7 批量操作（新增）
- 多选机制：`Ctrl+点击` 多选，`Shift+点击` 范围选择。
- 选中行高亮显示。顶部出现悬浮批量操作工具栏，提供：
  - 批量完成
  - 批量修改颜色
  - 批量删除（带确认）
- 多选模式不影响正常单击/双击操作，仅在按下修饰键时激活。

### F8 任务备注（轻量扩展）
- 支持为每条待办附加一段纯文本备注。
- 呼出方式：右键任务文字 → “添加备注”，或点击行尾的 `…` 图标（悬停显示）。
- 备注以小号文字显示在任务下方，不占用主视图高度；可折叠。
- 数据存储在 JSON 中，不影响虚拟滚动性能。

### F9 快捷键系统
| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+T` | 全局唤起/隐藏主窗口（托盘态恢复并置顶） |
| `Esc` | 关闭主窗口（最小化至托盘） |
| `Ctrl+Z` | 撤销最近一次完成操作 |
| `Ctrl+N` | 聚焦新建输入框 |
| `Ctrl+W` | 关闭当前展开的备注编辑面板 |
- 所有快捷键支持在设置中自定义。

## 三、交互规则（悬停 & 视觉反馈）

| 规则 ID | 描述 |
|----------|------|
| IR1 | 拖拽手柄（⋮⋮）、完成图标（✓）、备注图标（…）默认可见度为隐藏。 |
| IR2 | 鼠标悬停到待办行时，该行的左侧手柄、右侧完成图标、备注图标同时平滑浮现（opacity 过渡 150ms）。 |
| IR3 | 悬停时整行背景变为极淡主题色 (rgba 变化)，作为视觉反馈。 |
| IR4 | 穿透模式下自动显示所有操作图标（因悬停无法触发），同时行背景保持半透明可见状态，确保可操作性。 |

## 四、桌面集成特性

### D1 窗口置顶
- 通过托盘菜单或窗口右键菜单切换置顶 / 取消置顶。置顶时窗口始终位于所有窗口之上。

### D2 固定在桌面
- 窗口不随 `Win+D` 最小化，始终显示在桌面壁纸之上、桌面图标之下。
- 实现方式：使用 `SetParent` 绑定到 `Progman` 或 WorkerW，并通过 `with_skip_taskbar` 属性隐藏任务栏图标。
- 若无法完全规避所有显示异常，设置中注明“已知限制：某些全屏应用可能覆盖”。

### D3 鼠标穿透
- 托盘右键菜单提供“鼠标穿透”开关。
- 开启后整个窗口区域对鼠标透明（点击可穿透操作桌面），但保留标题栏区域（高度 32px）不穿透，允许用户关闭穿透。
- 穿透状态下窗口边框显示 1px 虚线作为视觉提示。
- 开启穿透时自动激活 IR4 规则，所有操作图标强制显示。
- 增加临时恢复交互：鼠标移至标题栏下方 20px 热区暂停穿透 2 秒，可进行快速操作。

### D4 透明度调节
- 设置面板提供背景透明度滑块（10%–100%），并预设三档：全透 (10%)、半透 (50%)、不透明 (100%)。
- 背景透明度仅改变窗口背景及亚克力模糊效果，文字和图标始终保持清晰。

### D5 系统托盘
- 关闭窗口（X 按钮或 `Esc`）不退出程序，窗口隐藏并最小化到系统托盘。
- 托盘图标始终可见，右键菜单包含：
  - 显示窗口
  - 置顶开关（勾选状态）
  - 鼠标穿透开关
  - 退出程序
- 双击托盘图标恢复窗口并置顶显示。
- 最小化时销毁前端渲染进程，仅保留托盘线程和 Rust 后端，内存降至 10MB 以下。退出态保存界面展开状态、滚动位置等，恢复后精确还原。

### D6 开机自启
- 设置内提供开关“开机自动启动”，开启后将程序注册为用户登录自启项，静默启动至托盘。

## 五、历史记录

### H1 已完成列表
- 独立面板，由主窗口“历史”按钮或托盘菜单打开。
- 按完成时间倒序排列，分组显示（今天、昨天、本周、更早）。

### H2 时间戳
- 每条历史记录显示“创建时间”和“完成时间”，格式为 `yyyy-MM-dd HH:mm`。

### H3 恢复为待办
- 每条已完成任务旁设有“恢复”按钮，点击后本条任务移回待办列表末尾，清除完成时间戳，子任务也一并恢复。
- 同时支持多选恢复。

### H4 清空 / 归档
- 提供“清空全部已完成”按钮（需二次确认）。
- 提供“归档为文件”按钮，将已完成数据导出为一个带时间戳的 JSON 文件至用户文档目录，并清空面板。

## 六、数据与可靠性

### DT1 本地存储
- 所有待办和历史数据存储于单个本地 JSON 文件，路径：`%APPDATA%/DeskTodo/data/todos.json`。
- 编码 UTF-8，格式化保存。

### DT2 自动备份（增强）
- **定时备份**：应用运行时每 5 分钟自动备份数据文件。
- **退出备份**：正常退出时立即备份。
- 备份文件保存至 `backup/` 子目录，命名：`todos_YYYY-MM-DD_HHmmss.json`。
- 采用“写入临时文件 + 重命名”原子操作，防止写半截文件。
- 仅保留最近 7 天的备份，超期自动删除。
- 启动时若检测到主数据 JSON 损坏或缺失，自动从最近备份恢复，并提示用户。

### DT3 撤销完成
- 任务完成后，提供 3 秒的轻量 toast 撤销入口（如前所述），同时支持 `Ctrl+Z` 撤销。

### DT4 数据导入/导出
- 设置界面提供“导出全部数据”按钮，将当前所有待办和历史导出为一个 JSON 文件，允许用户选择保存路径。
- 提供“导入数据”按钮，支持将之前导出的 JSON 文件合并到现有数据中（相同 ID 覆盖，不同 ID 新增），或替换全部数据（需确认）。
- 导入前自动校验 JSON 格式。

### DT5 界面状态持久化
- 每次退出或进入托盘前自动保存：
  - 已展开的父任务 ID 列表
  - 滚动位置（scrollTop）
  - 窗口位置与大小（按显示器记录）
- 恢复时还原这些状态，保证无缝连续。

## 七、UI 与性能要求

| ID | 要求 |
|----|------|
| U1 | 视觉风格严格遵循 Win11 设计语言：亚克力模糊背景、8px 圆角、Segoe UI Variable 字体（备选回退到系统默认）、极简阴影。 |
| U2 | 资源占用：正常运行内存 30-80MB，托盘态 ≤10MB；CPU 空闲时接近 0%。 |
| U3 | 虚拟滚动：待办列表使用 @tanstack/react-virtual，仅渲染视口内行，支持动态行高（含备注展开）。 |
| U4 | 安装包体积 ≤ 5MB，便携版 ZIP 解压后文件数少，可直接运行。 |
| U5 | 完整支持暗黑/明亮模式，跟随系统或手动切换，亚克力参数相应调整。 |
| U6 | 多显示器与高 DPI 适配：窗口位置记忆屏幕编号与坐标，不出现模糊或错位。 |

## 八、技术实现要点摘要

| 模块 | 技术方案 |
|------|----------|
| 窗口管理 | Tauri 2 无边框窗口，Rust 端控制置顶、穿透、托盘、自启。 |
| 前端框架 | React 18 + TypeScript，Zustand 管理全部状态。 |
| 样式系统 | Tailwind CSS 按需编译，结合 CSS 变量实现主题切换。 |
| 虚拟滚动 | @tanstack/react-virtual，预估行高 40px，动态测量。 |
| 数据持久化 | Rust 侧 `tauri-plugin-store` 或自定义文件读写，保证原子写入。 |
| 托盘与后台 | `tauri-plugin-tray`，关闭时销毁 WebView 释放内存。 |
| 桌面固定 | Rust 调用 `SetParent` 绑定窗口到桌面 WorkerW。 |
| 鼠标穿透 | `window.set_ignore_cursor_events(true)`，标题栏区域单独处理。 |
| 开机自启 | `tauri-plugin-autostart`。 |
